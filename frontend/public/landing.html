02 — Step by Step
How it works

From the first login attempt to automatic IP block — the complete journey.

01
Real log data comes in
Anomalyze watches real log files (Linux auth.log, Nginx/Apache access logs, Windows Event Log) using Python watchdog. New entries are detected the moment they're written. In DEV_MODE a synthetic generator produces realistic attack traffic automatically.
May 10 03:21:47 sshd[1234]: Failed password for root from 45.33.32.156 port 4821 ssh2
02
Preprocessor extracts 10 ML features
Raw log strings are parsed with regex into a fixed 10-dimensional float vector: status code, bytes, request length, failure/success/suspicious flags, hour of day, status category, and log type. This standardised vector feeds both detection engines.
features = [0.0, 0.0, 32.0, 1.0, 0.0, 1.0, 3.0, 4.0, 1.0, 0.0]
03
Rule engine checks 10 known attack patterns
10 explicit heuristic rules fire instantly for known patterns. IP-based sliding window counters track multi-event attacks like brute force (5+ failures/5min) and DoS (50+ req/min). Every rule maps to a MITRE ATT&CK technique.
if failed_attempts[ip] >= 5 in 5min → BRUTE_FORCE_DETECTED HIGH (T1110)
if request_count[ip] >= 50/min → DOS_ATTACK_DETECTED HIGH (T1498)
if SQLI_PATTERN.search(raw) → SQL_INJECTION HIGH (T1190)
04
Isolation Forest ML catches unknown anomalies
Even if no rule fires, the Isolation Forest model (200 trees, sklearn Pipeline with StandardScaler) compares the feature vector against normal baseline traffic. Score above 40% triggers an ML_ANOMALY_DETECTED alert. Model auto-retrains every 500 new normal samples via APScheduler.
05
Alert created + WebSocket broadcast + response
Alert Manager stores the record in SQLite with severity, MITRE technique, and ML score. WebSocket broadcasts the event to all connected dashboard tabs instantly — sub-100ms latency. HIGH/CRITICAL triggers iptables IP blocking (Linux) or netsh (Windows) and Gmail email alert simultaneously.

06 — Common Questions
Frequently Asked Questions

Everything you need to know before deploying or using Anomalyze.

?
What exactly does Anomalyze do and who is it for?
Anomalyze is a security platform that watches your server's activity logs around the clock and alerts you the moment something suspicious happens — a brute force login attack, SQL injection attempt, or unusual traffic patterns the AI hasn't seen before. It's for system administrators, IT teams, and security analysts who want 24/7 automated protection without hiring a full security team.
?
Do I need to be a cybersecurity expert to use this?
No. The dashboard is designed so anyone with basic IT knowledge can understand what's happening. Alerts show plain-English descriptions like "SSH Brute Force — 6 failed login attempts from 45.33.32.156". The system automatically blocks attackers and sends email notifications. The MITRE ATT&CK IDs and investigation guides are there for analysts who want deeper context.
?
How is this different from a regular antivirus or firewall?
Antivirus scans files for known malware signatures. Firewalls block specific ports or IPs based on manual rules. Anomalyze reads your actual server activity logs and uses AI to understand behavior patterns. It can catch an attacker who is slowly trying passwords over several hours, or detect an unusual API call pattern that no signature database would flag. It also acts as a SOAR platform — it doesn't just detect, it responds automatically.
?
What happens when Anomalyze detects something dangerous?
Immediately and automatically: (1) An alert is created with severity classification (HIGH/MEDIUM/LOW). (2) For HIGH threats — the attacker's IP is blocked via iptables (Linux) or netsh (Windows). (3) A notification is sent via Gmail SMTP email and/or Slack Webhook. (4) The dashboard updates in real time via WebSocket — no refresh needed. All of this happens within milliseconds of detection, with zero human intervention required.
?
What does "AI detection" actually mean here?
The AI component is Isolation Forest — a machine learning algorithm trained on what "normal" traffic looks like on your system. It works by randomly partitioning activity data and measuring how many cuts are needed to isolate each data point. Abnormal events get isolated in very few cuts — the algorithm assigns a risk score from 0–100%. Anything above 40% triggers an ML anomaly alert. The model auto-retrains every 500 new normal samples via APScheduler.
?
What are the two login roles and what can each do?
ADMIN — full access: view all alerts, resolve/delete them, trigger manual IP blocks, configure detection rules, retrain the ML model, manage user accounts, test notifications. ANALYST — read-only: view dashboard, browse alerts, view alert details and MITRE investigation guides, export CSV reports. Aria (the AI chatbot) also behaves differently per role — giving Admins deeper analytical capability.
?
Can I run this on my own company's servers?
Yes — Anomalyze runs entirely on your own infrastructure. Install on any machine with Python 3.11 and Node.js. It monitors your server's own log files directly — auth.log, nginx, apache, Windows Event Log. All your data stays on your hardware. The only external calls are GeoIP (ip-api.com, free, no key) and email alerts to your own Gmail account. Run backend with uvicorn, frontend with npm, open at localhost:5173.
?
Does it cost anything to run?
No. The entire stack is 100% free and open-source. Python, FastAPI, React, SQLite, Scikit-learn — all free. GeoIP (ip-api.com) is free with no key. Email alerts use Gmail's free SMTP. AbuseIPDB has a free tier of 1,000 checks/day. There are no subscription fees, no cloud costs, no API payments. You run it on your own hardware — a basic VPS or spare PC is sufficient.
